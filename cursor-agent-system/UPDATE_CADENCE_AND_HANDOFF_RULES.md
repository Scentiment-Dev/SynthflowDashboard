# Update Cadence and Handoff Rules

## Work-block updates

Each agent provides an update after a meaningful work block, before switching tasks, or before handoff.

## Required update format

```text
Completed:
Changed files:
Checks run:
No-drift status:
Blockers:
Next:
```

## Handoff timing

A handoff is required when the agent finishes a task, is blocked, changes ownership area, opens a PR, is ready for review, or discovers a no-drift concern.
